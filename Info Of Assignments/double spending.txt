# Theory

In blockchain systems, transactions occur between digital wallets. Every wallet has a balance, and transactions transfer cryptocurrency from one wallet to another.

One major problem in digital currency systems is **double spending**, where the same digital coins are spent more than once. Blockchain prevents this by validating transactions and maintaining transaction history.

This program simulates wallet transactions using Python dictionaries and checks whether:

* The sender has enough balance.
* The transaction ID has already been used.

If the transaction ID already exists, the program detects double spending and rejects the transaction.

# Explanation of Code

The program creates three wallets: Alice, Bob, and Charlie with initial balances. A transaction history list is maintained to store processed transaction IDs. 

The `send()` function simulates a transaction between wallets. First, it checks whether the transaction ID already exists in the transaction history. If it exists, the program detects double spending and rejects the transaction.

Next, the program checks whether the sender has sufficient balance. If the balance is enough, coins are transferred from sender to receiver and the transaction ID is stored in transaction history.

The program then demonstrates:

* A successful transaction
* A double-spending attempt
* A transaction failure due to insufficient balance

Finally, the updated balances of all wallets are displayed.
