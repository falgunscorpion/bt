# Theory

A token smart contract is a blockchain program that manages digital tokens. Ethereum tokens are generally created using Solidity and deployed on the Ethereum blockchain.

In this assignment, a Solidity token contract is compiled and deployed using Python and Web3.py. After deployment, tokens are transferred from the deployer account to another account.

The smart contract maintains balances for accounts and updates balances during token transfers. Transactions are signed using the sender’s private key and broadcasted to the blockchain network.

The assignment demonstrates:

* Smart contract deployment
* Token balance management
* Blockchain transaction execution
* Interaction between Python and Ethereum

# Explanation of Code

The program deploys a Solidity token smart contract and transfers tokens between Ethereum accounts using Python and Web3.py. 

First, environment variables such as RPC URL, private key, sender address, and receiver address are loaded securely using dotenv. The program then connects to the Ethereum blockchain using Web3 HTTP provider.

The Solidity token contract (`SimpleToken.sol`) is compiled using Solidity compiler version 0.8.20. Compilation generates ABI and bytecode required for deployment.

Next, a contract deployment transaction is created using an initial token supply. The deployment transaction is signed using the sender’s private key and broadcasted to the blockchain.

After successful deployment, the contract object is created using the deployed contract address and ABI. The balances of sender and receiver accounts are checked using the `balanceOf()` function.

A token transfer transaction is then created using the `transfer()` function. The transaction is signed, sent to blockchain, and confirmed. Finally, updated balances are displayed to verify successful token transfer.
