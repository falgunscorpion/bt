# Theory

A Bitcoin block contains:

* Block header
* Transactions
* Merkle Root
* Previous block hash
* Timestamp
* Nonce

The Merkle Root is a single hash representing all transactions in a block. It is generated using a Merkle Tree, where transaction hashes are repeatedly combined and hashed until one final hash remains.

Merkle Trees help:

* Verify transactions efficiently
* Detect tampering
* Maintain blockchain integrity

In this assignment, transactions are read from a file, hashed using SHA-256, and combined recursively to compute the final Merkle Root.

# Algorithm

1. Read transactions from file.
2. Hash all transactions using SHA-256.
3. Pair transaction hashes.
4. Combine paired hashes.
5. Re-hash combined values.
6. Repeat until one hash remains.
7. Final hash becomes Merkle Root.

# Explanation of Code

The program computes the Merkle Root for sample Bitcoin transactions using SHA-256 hashing. 

First, transactions are read from a text file. Each transaction is hashed individually using SHA-256 through the `hash_data()` function.

The program then constructs a Merkle Tree by pairing adjacent transaction hashes and combining them. The combined hashes are hashed again to form parent nodes.

If the number of hashes is odd, the last hash is duplicated to maintain the binary tree structure. This process continues recursively until only one hash remains.

The final remaining hash is called the Merkle Root, which represents all transactions in the block.
