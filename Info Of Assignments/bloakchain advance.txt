# Theory

Blockchain is a secure distributed ledger where blocks are connected using cryptographic hashes. Every block contains its own hash and the hash of the previous block.

Blockchain verification is the process of checking whether:

* The current hash of each block is valid.
* The previous hash stored in a block matches the hash of the previous block.

If any block data changes, the hash changes completely, breaking the blockchain integrity.

The verification function ensures:

* Data integrity
* Tamper detection
* Security of blockchain

---

# Important Concepts

## 1. Blockchain Integrity

Ensures that blockchain data has not been modified.

## 2. Hash Verification

Checks whether stored hash matches recalculated hash.

## 3. Previous Hash Validation

Verifies linking between consecutive blocks.

## 4. Immutability

Data cannot be altered without affecting the entire blockchain.

## 5. Tampering Detection

Modification in block data invalidates the blockchain.

---

# Hashing Concept

Hash = SHA256(Data + Timestamp + Previous_Hash)

---

# Algorithm

1. Create blockchain with linked blocks.
2. Traverse all blocks in the chain.
3. Recalculate hash of each block.
4. Compare recalculated hash with stored hash.
5. Verify previous hash linkage.
6. If mismatch found, blockchain is invalid.
7. Otherwise blockchain is valid.

---

# Flowchart

Create Blockchain → Traverse Blocks → Recalculate Hash → Compare Hashes → Verify Previous Hash → Validate Blockchain Integrity

# Explanation of Code

The program implements blockchain verification using Python. A blockchain is created where every block contains data, timestamp, previous hash, and its own hash generated using SHA-256.

A verification function is implemented to validate the blockchain. The function traverses all blocks and recalculates hashes for each block. It compares the recalculated hash with the stored hash.

The function also checks whether the previous hash stored in the current block matches the hash of the previous block. If any mismatch occurs, it indicates tampering and the blockchain becomes invalid.

This assignment demonstrates how blockchain ensures security and integrity through cryptographic linking and verification.
