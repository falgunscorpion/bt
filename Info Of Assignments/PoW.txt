# Theory

Proof-of-Work (PoW) is a consensus mechanism used in blockchain systems like Bitcoin to validate blocks and secure the network.

In PoW, miners solve a computational puzzle by repeatedly changing a value called a **nonce** until the generated hash satisfies a specific condition, usually starting with a certain number of leading zeros.

Mining helps:

* Secure blockchain
* Prevent fraud
* Validate transactions
* Achieve consensus

The mining process requires computational effort, making blockchain tampering extremely difficult.

# Algorithm

1. Create block data.
2. Initialize nonce to 0.
3. Generate hash using block data and nonce.
4. Check whether hash satisfies difficulty condition.
5. If condition fails, increment nonce.
6. Repeat until valid hash found.
7. Store mined block.

# Explanation of Code

The program simulates a simple blockchain mining process using Proof-of-Work. A block is created containing transaction data and a nonce value.

The mining function repeatedly generates SHA-256 hashes by changing the nonce value. The generated hash is checked against the difficulty level, which specifies the required number of leading zeros.

If the hash does not satisfy the condition, the nonce is incremented and hashing is repeated. Once a valid hash is generated, the block is considered mined successfully.

The program demonstrates how computational work is required to validate blocks in blockchain systems.
