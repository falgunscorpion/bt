# Theory

A digital wallet is a software application used to store and manage cryptocurrency keys securely. It uses asymmetric cryptography, where two mathematically related keys are generated:

* Public Key
* Private Key

The private key is secret and used for authentication and digital signatures, while the public key is shared publicly to receive transactions.

The wallet address is generated from the public key. Blockchain wallets do not store actual cryptocurrency; instead, they store cryptographic keys used to access blockchain assets.

The assignment demonstrates the use of cryptography, hashing, and digital signatures to simulate wallet generation and secure transaction handling.

# Important Concepts

## 1. Private Key

A secret cryptographic key used to:

* Access wallet
* Sign transactions
* Prove ownership

The private key must never be shared.

---

## 2. Public Key

A public cryptographic key generated from the private key.

Used to:

* Receive cryptocurrency
* Verify digital signatures

---

## 3. Key Pair

Combination of:

* Public key
* Private key

generated mathematically together.

---

## 4. Digital Wallet

A software system that stores:

* Public key
* Private key
* Wallet address

---

## 5. Wallet Address

A shorter representation generated from the public key and used for receiving cryptocurrency.

---

## 6. Asymmetric Cryptography

Encryption technique using two different keys:

* Public key
* Private key

---

## 7. Digital Signature

A cryptographic signature created using the private key and verified using the public key.

---

## 8. Hashing

A process that converts data into a fixed-length secure value.

Common hashing algorithm:

* SHA-256

---

## 9. Elliptic Curve Cryptography (ECC)

A mathematical cryptographic method used for secure key generation in blockchain systems.

# Explanation of Code

The program generates a public and private key pair using cryptographic algorithms. First, a random private key is generated securely. Then, the public key is mathematically derived from the private key using elliptic curve cryptography.

The public key is further processed to create a wallet address. The wallet stores these keys securely and simulates digital wallet functionality.

The private key is used to digitally sign transactions, while the public key is used to verify signatures. Hashing algorithms such as SHA-256 are used to secure wallet data and generate unique identifiers.
